Yo Jekyll - my blogging platform

Sep 20, 2015 · 0 comments
2 minute read

I have been hearing and seeing developer & enthusiasts use Jekyll1, but never had a chance to check it out - until now! And I must say that I love the concept, simplicity and git. It sure does take some time to understand Jekyll, but in the end its all worth the effort!

A Basic Jekyll Site

On researching and looking at various blogs patronizing Jekyll & the cool-new way of blogging, I wanted to give it a shot. I initially started off with testing a basic Jekyll blog using

 ~ $ gem install jekyll
 ~ $ jekyll new my-awesome-site
 ~ $ cd my-awesome-site
 ~/my-awesome-site $ jekyll serve

This popped up a basic site on http://localhost:4000. And I already feel so good! The same feeling when I endorsed LaTeX2

I generally started reading more on Jekyll and how concepts like _posts, pages, templates, etc. works. Then I stumbled upon a post by David Calhoun3, on using Yeoman4 a.k.a Yo, a web scaffolding tool that helps you kickstart a project with Jekyll. Things just got interesting!

Its not very tough to kickstart a Jekyll blog using Yeoman:

Install Jekyll with Yeoman

 ~ $ npm install -g yo
 ~ $ npm install -g generator-jekyllrb
 ~ $ mkdir ~/Sites/myblog && cd $_
 ~ $ yo jekyllrb

These were some of my responses to the generator:

Wire tools and preprocessors.

? CSS preprocessor: Compass
? Use Autoprefixer? Yes
? Javascript preprocessor: None

Choose a template.

? Site template: Default Jekyll

Choose deployment options.

? Use grunt-build-control for deployment? No

This builds (creates a static version of the source code that you have) and creates a server to view the website at http://localhost:3000. The best part here is that Yo comes with Browsersync - and that is awesome! I spent some more time playing around with the platform and started to get a hang of it.

Theming

I embrace simple design (as much as I can) and started looking for themes to help me get started. Lanyon, a theme based on Poole was an obvious winner! :)
