Modifying config / file in vendor project (via composer)

Feb 25, 2016 · 0 comments
2 minute read

Let me start by saying that I understand - its not a good idea / practice to change vendor files. But there are situations, where this cannot be avoided!

Recently when using mPDF package with CakePHP via composer at https://github.com/mpdf/mpdf, I needed to touch one of the config files in the package - specifically the config_fonts.php as there was a need to change or add default fonts1 when generating pdf.

One solution

One solution is to manually download the package and add it to your “Library” folder. This will unfortunately increase the size of your codebase!

My Solution

We can leverage the post-autoload-dump event in composer2 to generate / overwrite the config file (after first backing up the original file)! The only thing to be maintained in our repository was the config file to be overwritten.
